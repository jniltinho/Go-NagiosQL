# NagiosQL Integration with Nagios Core 4.5.13 Debian Packages

## Overview

This document describes how to integrate NagiosQL with a Debian-packaged Nagios Core 4.5.13 installation.

The objective is to:

* Keep Nagios Core managed by Debian packages
* Keep NagiosQL isolated from package-managed files
* Allow safe upgrades
* Support Nginx + fcgiwrap deployments
* Avoid configuration conflicts
* Follow Debian filesystem standards

---

# Recommended Architecture

```text
NagiosQL
    |
    v
/etc/nagios/generated/
    |
    v
/etc/nagios/nagios.cfg
    |
    v
Nagios Core
    |
    v
Nagios Plugins
```

The NagiosQL web interface generates configuration files.

Nagios Core consumes those files through `cfg_dir` directives.

# Validation Workflow

Never reload Nagios without validating configuration.

Create:

```text
/usr/local/bin/nagiosql-deploy.sh
```

Content:

```bash
#!/bin/bash

/usr/sbin/nagios -v /etc/nagios/nagios.cfg

if [ $? -eq 0 ]; then
    systemctl reload nagios
else
    echo "Configuration validation failed"
    exit 1
fi
```

Permissions:

```bash
chmod +x \
  /usr/local/bin/nagiosql-deploy.sh
```

---

# Systemd Integration

Create:

```text
/etc/systemd/system/nagios-reload.service
```

Content:

```ini
[Unit]
Description=Nagios Configuration Reload

[Service]
Type=oneshot

ExecStart=/usr/local/bin/nagiosql-deploy.sh
```

Reload systemd:

```bash
systemctl daemon-reload
```

# Nginx Integration

Recommended deployment:

```text
Nginx
  |
PHP-FPM
  |
NagiosQL
```

Example virtual host:

```nginx
server {

    listen 80;

    server_name nagiosql.example.com;

    root /var/www/nagiosql;

    index index.php;

    location / {
        try_files $uri $uri/ /index.php?$query_string;
    }

    location ~ \.php$ {

        include fastcgi_params;

        fastcgi_param SCRIPT_FILENAME $document_root$fastcgi_script_name;

        fastcgi_pass unix:/run/php/php8.4-fpm.sock;
    }
}
```

# Upgrade Procedure

## Upgrade Nagios Core

```bash
apt update

apt install nagios-core
```

No NagiosQL changes required.

---

## Upgrade NagiosQL

```bash
apt install nagiosql
```

Re-generate configuration:

```text
Tools
  -> Write Configuration Files
```

Validate:

```bash
nagios -v /etc/nagios/nagios.cfg
```

# Security Recommendations

* Use HTTPS only
* Restrict access with authentication
* Restrict NagiosQL administration access
* Backup configuration daily
* Validate before reload
* Use dedicated MariaDB credentials
* Avoid running NagiosQL as root
